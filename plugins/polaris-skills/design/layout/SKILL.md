# Layout

Layout determines the placement, positioning, and organization of various components within the UI: buttons, menus, content sections, and visual elements.

## Space Defines Proximity

Space plays a vital role in establishing connections between items. Proximity principle: closer objects have stronger perceived relationship.

**Grouping benefits:**
- Merchants understand which elements are related
- Faster decision-making
- Clear information architecture

**Do:**
- Group similar data points or tasks in the same card
- Nest inset shapes and surfaces
- Use space to show relationships

**Don't:**
- Scatter related information across UI
- Ignore spatial relationships

## Emphasis Creates Hierarchy

In the Shopify admin, visual prominence creates order.

**Visual weight hierarchy:**
- Larger, heavier, contrasting elements attract attention (primary)
- Smaller, lighter, subtle elements provide detail (secondary)

**Emphasis techniques:**
- Use size to command attention
- Use weight to establish sections
- Use contrast to create visual rhythm

**Divider usage:**
- Use divider lines in data and index tables
- Rarely use dividers elsewhere

**Do:**
- Use weight and contrast to introduce UI sections
- Establish clear visual hierarchy
- Size elements appropriately

**Don't:**
- Use divider lines for hierarchy outside tables
- Over-complicate with excessive dividers

## Software, Not Website

Admin is more like software than a website. Elements need sizing based on their job.

**Sizing principles:**
- Compact elements add detail and precision
- Larger elements command attention
- Surfaces adapt to components
- Optimize views for different tasks

**Do:**
- Create compact components for specialized, minute tasks
- Size elements appropriately for importance
- Match element scale to task importance

**Don't:**
- Contradict task importance with size
- Create confusing size relationships
- Over-enlarge minor elements

## Best Practices

- Establish clear visual hierarchy through size, weight, contrast
- Use space intentionally to show relationships
- Think in terms of information architecture
- Consider task importance in sizing decisions
- Maintain software-appropriate density
