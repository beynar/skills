# Depth

Depth introduces a sense of realism, helps establish visual hierarchy, and creates focus.

## Hierarchy

Depth effectively establishes visual hierarchy. The higher an element sits in the Z scale, the more important it will seem.

**Do:**
- Use depth tactically to differentiate between primary and secondary elements
- Guide merchants through understanding importance of different elements

**Don't:**
- Overuse depth (leads to clutter and confusion)
- Add unnecessary depth to elements
- Allow elements to protrude outside parent containers (disrupts depth perception)
- Break parent-child relationship integrity

## Interactivity

Depth indicates interactivity. Interactive elements (buttons, cards) get more depth to signal merchants can interact with them.

**Do:**
- Use depth to indicate interactive elements clearly
- Apply intuitive changes upon interaction (e.g., pushing button down)
- Provide visual feedback through depth changes
- Intensify sense of tactility with depth shifts

**Don't:**
- Give static elements unnecessary depth (misleads merchants)
- Apply unexpected depth changes (causes disorientation)
- Create confusion about what's interactive

## Focus

Depth helps guide merchant focus by directing attention to important elements.

**Do:**
- Use depth to highlight actions and large pieces of information overlaying others
- Ensure merchants see important elements first
- Layer information with intentional depth

**Don't:**
- Rely solely on depth to create focus
- Forget that not all merchants perceive depth the same way
- Skip alternative focus techniques for accessibility
- Use depth alone without combining with other techniques

## Accessibility

Use a combination of techniques to ensure design is accessible to everyone perceiving depth differently.
