# Illustrations

The Shopify admin uses a precise illustration style to help merchants quickly and clearly understand how things work across every experience.

## Principles

### Always Be Useful

Illustration adds information. It:
- Provides context and clarity
- Leads to the next step
- Gives merchants deeper understanding
- Supports the experience

**Do:**
- Every illustration should serve a purpose
- Add context or clarity
- Guide merchants forward

**Don't:**
- Use decorative illustrations
- Add illustrations without purpose

### Keep It in the Family

Illustrations are all part of the same visual family.

**Why consistency matters:**
- Inconsistencies lower overall quality
- Can distract merchants
- Makes merchants question if they're in the right place
- Creates cohesive experience

**Do:**
- Maintain visual harmony
- Follow established style
- Keep consistent approach

**Don't:**
- Create one-off unique styles
- Break visual family rules
- Over-stylize inconsistently

### Be Considerate

Illustration should understand and support the merchant experience.

**Considerations:**
- Appropriate for situation
- Supportive of experience
- Contextually correct
- Emotionally appropriate

**Do:**
- Understand the context
- Support merchant journey
- Make illustrations appropriate

**Don't:**
- Use out-of-place illustrations
- Create confusing context

### Stay Focused

Each illustration conveys one thing clearly.

**Do:**
- Tell single story
- Make it easy to understand
- Help merchants intuitively know what to do
- Focus on one concept

**Don't:**
- Overload with multiple messages
- Create confusing narratives
- Show too much

## Elements of Style

### Color

Illustrations use special limited color palette.

**Color approach:**
- Whites, grays, two or three colors per illustration
- Colors less saturated than surrounding UI
- Prevents distraction from core interactions
- Designed to work in specific contexts

**Do:**
- Use limited palette
- Less saturation than UI
- Maintain visual balance

**Don't:**
- Over-saturate colors
- Compete with UI for attention

### Shape

Objects have realistic proportions for easy recognition.

**Shape principles:**
- Simple geometric shapes with rounded corners
- Clear and approachable
- Organic shapes for people
- Easy to recognize
- Realistic proportions

**Do:**
- Use simple geometric shapes
- Round corners
- Realistic proportions
- Clear recognition

**Don't:**
- Over-complicate shapes
- Use unrealistic proportions
- Create ambiguity

### Space

Perspective is flat and two-dimensional.

**Space considerations:**
- Flat perspective, equal visual importance
- Drop shadows for subtle depth when necessary
- Additional sides of objects for clarity (2.5D views)
- Negative space around illustration
- Balanced visual weight
- Consistent with other illustrations

**Do:**
- Use negative space
- Keep balanced weight
- Add sides for clarity if needed
- Maintain consistent spacing

**Don't:**
- Crowd illustrations
- Create perspective confusion
- Overuse depth effects

### Line

Line makes and arranges shapes in space.

**Line characteristics:**
- Smooth lines without texture
- Smaller objects = straighter lines
- Larger objects = more detailed, curved lines
- Intersecting and continuous lines create elegance
- Avoid forced or artificial line patterns

**Do:**
- Use smooth, clean lines
- Vary line style by object size
- Create visual interest with line
- Use intersecting lines naturally

**Don't:**
- Create textured or rough lines
- Force line patterns
- Over-detail line work

### Detail

Illustrations need some detail to make sense, but not too much.

**Detail principles:**
- Minimum necessary detail
- Realistic but simple
- Fine details rarely smaller than 4px
- Avoid noise and clutter
- Support recognition

**Do:**
- Include essential detail
- Keep simple
- Support clarity

**Don't:**
- Over-detail
- Create noise
- Include unnecessary complexity

## Where Illustrations Live

### Empty States

Merchants see these first time accessing new areas.

**Purpose:**
- Introduce what they can do here
- Set expectations
- Frame the experience

**Use for:**
- First-time experiences
- Onboarding new features
- Setting context

### Onboarding

Help new merchants set up their store.

**Purpose:**
- Frame what each task is for
- Reinforce completion (changing appearance)
- Guide setup process

**Use for:**
- Setup tasks
- Initial configuration
- Learning new features

### Announcements

Let merchants know about helpful things.

**Purpose:**
- Celebrate milestones
- Introduce important products
- Make announcements special and noticeable

**Use for:**
- Major merchant milestones
- Important product launches
- Strategic communications

### Spot Illustrations

Rare use for specific goals.

**Purpose:**
- Draw attention to important items on busy pages
- Explain technical concepts
- Create focal points

**Use for:**
- Complex explanations
- Important highlights
- Specific technical concepts

## Design Process

1. Start with clear purpose
2. Research context and situation
3. Sketch simple representations
4. Iterate within style system
5. Test for clarity and context fit
6. Ensure family consistency
7. Verify detail level
8. Check color appropriateness
