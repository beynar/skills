# New Features

Help merchants discover new and impactful features.

## When to highlight new features

Features should have **all 3** of these characteristics before highlighting with a New Badge or PIP:

1. **High business value**: Shopify wants to drive adoption because it delivers significant value
2. **Outsized value for merchants**: Creates new value, not just an improved way to do something they're already doing
3. **Worth distracting from workflow**: Important enough to pull merchants' attention from their current task

**Things that generally meet this criteria** are large and impactful. Examples: entire new pages in settings or introduction of entirely new technology or capability.

## What to avoid

### Inconsistency
Different teams using different methods to highlight features leads to inconsistent merchant experience.

### Visual clutter
Over time, numerous badges and pips make the admin UI cluttered and difficult for merchants to focus on primary tasks.

## Ways to bring feature awareness

### New Badge

- Use the informational badge variant for correct styling and color
- Place badge right-aligned or to the right of text
- The Page component in Polaris already places badges to the right of headings—follow this logic for consistency

### Pips

A pip can highlight a status or new element in a list. Example: next to a new notification in a notification list. Should NOT be used for representing new features.

### Duration of visibility

New Badges and PIPs should have a short lifespan. The badge should disappear when:

- The user has clicked on the interactive element it's attached to, OR
- 5 days have passed since they first saw it, OR
- After 3 sessions (landing on a page 3 times)

Choose the first condition achieved.
